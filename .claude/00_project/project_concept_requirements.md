# 秩父三十四箇所巡礼アプリ 要件定義書

## プロジェクト概要

埼玉県秩父市に存在する34ヶ所の札所を巡る際に使用するモバイルアプリケーション

## 基本方針

- 完全無料のライブラリを使用してコストを最小限に抑える
- Expo (React Native) をベースに開発

---

## 機能要件

### 1. 主な利用シーン

- 実際に巡礼中にスマホで使う（リアルタイムナビゲーション）
- 事前の計画・下調べに使う（情報閲覧）
- **両方のシーンに対応**

### 2. 地図機能

#### 2.1 地図表示

- 全34ヶ所の札所の位置にピンなどの目印を表示
- 現在地周辺の札所を表示
- ズームイン・ズームアウトなどの地図操作が可能
- 地図から各札所の場所を確認できる

#### 2.2 経路表示機能

- 現在位置から選択した札所までの経路を表示
- **徒歩ルート**に対応
- **車のルート**に対応

### 3. 札所一覧機能

- 34ヶ所の札所をリスト表示
- わかりやすい一覧形式

### 4. 札所情報

各札所について以下の情報を表示：

- 札所番号
- 札所名称
- 住所

### 5. 巡礼記録機能

- 訪問済みの札所をチェックできる機能
- チェック状態の保存

---

## 技術要件

### フレームワーク・ライブラリ

- **フレームワーク**: Expo (React Native)
- **ルーティング**: Expo Router（ファイルベースルーティング）
- **地図ライブラリ**: react-native-maps（無料、ネイティブ地図）
- **経路検索API**: OSRM (Open Source Routing Machine)（完全無料）
- **スタイリング**: StyleSheet（React Native 標準）
- **アイコン**: @expo/vector-icons

### データ管理

- **札所データ**: 静的JSONファイルとして管理
- **訪問記録**: AsyncStorage（デバイスローカル保存）

### デプロイ

- **推奨**: EAS (Expo Application Services)（Expoと相性が良く、無料枠あり）

---

## 非機能要件

### データ保存

- 訪問記録などは個人のデバイス内のみで保存
- 複数デバイス間での同期は不要

### オフライン対応

- オフライン対応は考慮しない

### データソース

- 札所の詳細情報（名称、番号、住所、座標）は別途調査・準備が必要
